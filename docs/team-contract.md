## 2. Jadwal Pertemuan (Meeting Schedule)

Pertemuan rutin dilakukan untuk sinkronisasi progress:

* **Hari**: Setiap weekend
* **Waktu**: 19.00 WIB - Selesai
* **Platform**: Google Meet / Discord / Offline
* **Agenda**: Update progress mingguan dan penyelesaian kendala teknis.

## 3. Saluran Komunikasi (Communication Channels)

* **Komunikasi Harian**: WhatsApp Group untuk koordinasi cepat dan pengumuman penting.
* **Diskusi Teknis**: Discord atau Zoom untuk diskusi mendalam terkait pengembangan dan permasalahan teknis.
* **Manajemen Tugas**: Spreadsheet Google Sheets untuk tracking progress, dan deadline.

## 4. Standar Pesan Commit (Commit Message Standards)

Gunakan format **imperative** untuk setiap perubahan di repositori:

* **Format**: `[Tag]: [Short Description]`
* **Contoh Tag**: `feat` (fitur baru), `fix` (perbaikan bug), `docs` (dokumentasi), `style` (formatting).
* **Contoh Pesan**:
    * `feat: add login validation logic`
    * `fix: resolve null pointer exception in database connection`
    * `docs: update team contract file`

## 5. Mekanisme Eskalasi (Escalation Mechanism)

1. **Diskusi Internal**: Masalah dibahas terlebih dahulu dalam pertemuan tim secara terbuka.
2. **Deadlock/Konflik**: Jika terjadi jalan buntu, ketua tim akan mengambil keputusan akhir.
3. **Bantuan Eksternal**: Jika masalah tetap tidak teratasi, tim akan segera menghubungi **Asisten Praktikum** sebagai mediator/pemberi solusi.
